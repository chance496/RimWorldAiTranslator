RimWorld AI Translator
======================

Version 1.0.0

RimWorld 모드를 프로젝트별로 불러와 AI 초벌 번역, 수동 번역, 검수,
업데이트 추적, Languages\Korean 적용과 RMK 로컬 작업까지 처리하는 Windows용 도구입니다.

실행
----
1. RimWorldAiTranslator.exe를 실행합니다.
2. 자동 검색된 모드를 선택하거나 모드 폴더를 직접 지정합니다.
3. 원문 언어가 여러 개면 프로젝트에서 사용할 기준 언어를 선택합니다.
4. 프로젝트를 열고 원문 갱신을 누릅니다.
5. 설정에서 Cerebras, OpenAI, Gemini, DeepSeek, Qwen, Groq, Mistral, OpenRouter,
   BigModel / Z.AI 또는 사용자 지정 API를 선택하고 키를 입력합니다.
6. 선택한 제공자의 API 키가 없으면 Google 번역 후보를 생성합니다.
7. 기존 번역이 있으면 덮어쓰기, 미번역 부분만 번역하기 또는 취소를 선택합니다.
8. 번역문을 확인하고 번역됨 또는 검토 완료 상태로 지정합니다.
9. RMK에 적용을 끄면 원본 모드에, 켜면 RMK 작업 클론에 기록합니다.
10. 검토 적용은 검토됨만, 전체 적용은 번역됨과 검토됨을 함께 반영합니다.

주요 기능
---------
- 프로젝트 하나당 모드 하나를 연결합니다.
- Steam Workshop과 로컬 RimWorld 모드를 자동으로 검색합니다.
- 원본 언어를 감지하고 여러 언어가 있으면 프로젝트 생성 때 기준 언어를 선택합니다.
- LoadFolders.xml의 현재 버전 폴더와 선택적 연동 Def도 함께 읽습니다.
- 선택한 제공자의 여러 API 키를 순환 사용합니다. Cerebras는 무료 API 한도를 기본 적용합니다.
- 기존 번역을 전체 재번역하거나 미번역 부분만 번역할 수 있습니다.
- 번역 전에 프로젝트, 원문 언어, 제공자·모델, 대상 수·배치와 예상 입력 토큰을 확인합니다.
- 실제 원문 분석·번역 단계와 배치 진행을 작업 화면 위 상단 상태 막대에 표시하고 오류 재시도, 취소와 완료분 복구를 제공합니다. 성공·취소 상태는 자동으로 닫힙니다.
- 프로페셔널, 사이파이, 비비드, 스튜디오, 프런티어 디자인 컨셉을 밝음·어두움과 조합해 선택할 수 있습니다.
- RMK, 모드 기존, AI 초벌, 내 번역 출처를 구분하고 내 번역 시각순으로 정렬합니다.
- 미번역, 번역됨, 검토됨, 변경됨 상태와 텍스트, 키, Def Class, Node 검색을 제공합니다.
- 품질 센터에서 미번역, 원문 변경, 토큰·태그, 길이와 중복 문제를 필터링하고 해당 문자열로 이동합니다.
- 비교 탭은 기존/AI 번역과 현재 편집의 변경 구간을 강조하며, 본문과 키를 제외한 HTML 품질 보고서를 저장할 수 있습니다.
- 빈 번역, 원문 변경, 주의 항목을 제외하고 전체 검토 완료로 일괄 표시할 수 있습니다.
- 같은 원문을 수동 번역하면 동일 원문 전체에 적용할지 확인합니다.
- 원문 갱신 후에도 모드에 아직 적용하지 않은 번역과 메모를 유지합니다.
- 같은 키의 원문이 바뀌면 기존 번역을 보존하고 변경됨 + 미번역으로 표시합니다.
- RMK 구독본의 기존 번역을 재사용하고, 없는 키만 AI 번역할 수 있습니다.
- RMK XLSX의 번역 당시 원문과 현재 같은 언어 원문을 비교하며, XLSX가 없으면 직전 프로젝트 원문을 Def Class + Node로 비교해 변경됨 + 미번역으로 표시합니다.
- 원문 갱신은 별도 프로세스로 실행되어 큰 모드에서도 창과 중지 버튼이 응답합니다.
- EXE는 클릭이 필요 없는 준비 화면과 가벼운 무한 진행 표시를 먼저 보여 주고, 테마·레이아웃·프로젝트 상태가 모두 준비된 메인 창으로 한 번에 전환합니다. 사전 컴파일된 네이티브 판독기로 큰 Def와 RMK XLSX를 읽습니다.
- 프로젝트 전환과 새 원문 결과 로드는 전체 내용을 준비한 뒤 한 번에 표시하므로 항목이 하나씩 나타나지 않습니다. 새 프로젝트는 분석 중 대시보드를 유지하고 성공한 뒤 완성된 검수 화면으로 전환합니다.
- 원문 분석 상태 막대는 화면 크기를 바꾸지 않으며, RMK 인덱스에 없는 모드는 느린 전체 폴더 탐색 없이 바로 로컬 프로젝트로 진행합니다.
- 검수 목록은 전체 문자열을 가상 목록으로 제공하고 화면에 보이는 행만 그려 큰 프로젝트도 빠르게 엽니다.
- 검수 상태는 AI/로컬 번역과 실제 변경 사항만 압축 저장해 시작과 자동 저장의 반복 작업을 줄입니다.

상태와 적용
-----------
- 미번역: 번역이 없거나 업데이트 후 다시 확인해야 합니다.
- 번역됨: 번역은 있지만 최종 검토 전입니다.
- 검토됨: 사람이 확인했습니다.
- 변경됨: 같은 키의 원문 값이 바뀌었습니다. 다시 처리하기 전에는 적용되지 않습니다.

검토 적용은 검토됨 항목만 기록합니다.
전체 적용은 검토됨과 안전한 번역됨 항목을 함께 기록합니다.

RMK 로컬 연동
-------------
- Steam Workshop RMK 구독본은 기존 번역 참조에만 사용하며 수정하지 않습니다.
- RMK에 적용을 체크하지 않으면 기존처럼 원본 모드의 Languages\Korean에 씁니다.
- 체크하면 설정에서 지정한 RMK bus 브랜치 작업 클론에 키 기준으로 병합합니다.
- 기존 키는 원래 RMK XML에서 수정하고 새 키만 추가한 뒤 LoadFoldersBuilder를 실행합니다.
- 림추출기 호환 XLSX를 함께 생성·갱신하며 기존 스타일, 주석, 추가 열과 Required Mods는 보존합니다.
- 아직 재검토하지 않은 변경 행은 XLSX의 과거 원문을 유지해 다른 작업자도 업데이트를 감지할 수 있습니다.
- 원문 변경 또는 안전 검사 실패 항목은 내보내지 않습니다.
- RMK 언어 폴더 기록은 현재 같은 언어 폴더와 비교합니다. Def에 직접 들어 있는 원문은 실제 Def 값과 비교하므로 Def 원문 언어 변경도 업데이트로 표시됩니다.
- Git 커밋과 푸시는 자동으로 하지 않습니다.

동일 원문 일괄 번역
-------------------
직접 입력한 번역을 저장하거나 다른 항목으로 이동할 때 같은 원문이 있으면
일괄 적용 여부를 확인합니다. 줄바꿈을 제외하고 원문이 완전히 같을 때만 묶습니다.
메모는 복사하지 않으며, 바뀐 다른 항목은 번역됨 상태로 돌아가 다시 검토할 수 있습니다.

주요 단축키
-----------
F2 번역문 입력 / F3 다음 / Shift+F3 이전
Alt+1 AI 후보 / Alt+2 기존 번역 / Alt+0 되돌리기
Ctrl+1 미번역 / Ctrl+2 번역됨 / Ctrl+3 검토 완료
Ctrl+Shift+3 안전한 번역 전체 검토 완료
Ctrl+Enter 검토 완료 후 다음 / Ctrl+F 검색 / Ctrl+S 저장
Ctrl+Shift+P 명령 팔레트 / Ctrl+Home 프로젝트 목록
Alt+C 선택 문자열 비교 / Alt+Q 프로젝트 품질 센터
F5 원문 갱신 / F9 AI 번역 / Shift+F9 중지 / F6 작업 영역 이동

API와 개인정보
-------------
- 기본 선택: Cerebras gemma-4-31b
- 제공자별 API URL, 모델, Temperature는 설정에 저장되며 직접 수정할 수 있습니다.
- 기본 배치: 40개
- API 키는 파일, 프로젝트, 명령행, 로그에 저장하지 않습니다.
- GUI의 API 키는 번역 프로세스 환경 변수로만 전달하며 임시 인수 파일에도 넣지 않습니다.
- 번역 실행은 cmd.exe 셸을 거치지 않고 허용된 매개변수만 전달합니다.
- 프로그램을 다시 실행하면 API 키를 다시 입력해야 합니다.
- 번역 원문은 선택한 AI 제공자 또는 Google 외부 서버로 전송됩니다.

로컬 데이터
-----------
프로젝트와 검수 데이터는 다음 위치에 저장됩니다.

%LOCALAPPDATA%\RimWorldAiTranslator

- projects.json: 프로젝트와 모드 연결 정보
- reviews\...\review-decisions.json: 번역문, 상태, 메모, 원문 변경 기록
- settings.json: 화면, 번역 API URL·모델, RMK 설정. API 키는 저장하지 않습니다.

프로그램 ZIP이나 EXE를 업데이트해도 이 데이터는 유지됩니다.

설정의 "진단 번들 저장"은 문제 보고용 ZIP을 사용자가 고른 위치에 만듭니다. 원문, 번역문, 번역 키, API 키, 전체 경로와 원시 로그는 포함하지 않습니다.

요구 환경
---------
- Windows 10/11
- Windows 기본 PowerShell
- 네트워크 번역 사용 시 인터넷 연결
- 기본 기능에는 Python, Node.js, Git, 별도 .NET 개발 도구가 필요하지 않습니다.
- RMK에 적용하려면 RMK Git 클론과 Git이 필요합니다.

주의
----
- defName, XML 키, 클래스명, 텍스처 경로 같은 게임 식별자는 번역하지 않습니다.
- AlienRace 색상 채널과 비표시용 PawnRenderTreeDef 필드처럼 확실한 내부 식별자는 번역·검수 목록에서 제외하고 _TranslationAudit에 사유를 기록합니다.
- Patches XML은 게임이 패치를 모두 적용한 뒤의 대상과 리스트 키를 외부에서 확정할 수 없어 자동 번역하지 않습니다.
- r_logentry-> 같은 문법 접두어와 괄호형 플레이스홀더는 원문 그대로 보존되어야 적용할 수 있습니다.
- 동적 이름 뒤의 조사는 (은)는, (이)가, (을)를 형식으로 씁니다. 은(는), 이(가), 을(를)처럼 뒤집힌 표기와 변수 뒤의 고정 조사는 주의로 표시하고 적용하지 않습니다.
- DLL/C# 코드에 직접 작성된 화면 문자열은 XML 번역만으로 바뀌지 않을 수 있습니다.
- 외부 번역 서비스에 보내면 안 되는 비공개 문자열에는 네트워크 번역을 사용하지 마십시오.
- 배포 EXE는 코드 서명 인증서로 서명되지 않아 Windows 경고가 나타날 수 있습니다.
