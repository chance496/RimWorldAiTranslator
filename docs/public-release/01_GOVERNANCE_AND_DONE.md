# 01 — 운영 계약과 공개 품질 완료 조건

이 문서의 안전 규칙은 구현 편의보다 우선한다.

## 1. 외부 작업 금지

사용자가 현재 대화에서 대상과 작업을 정확히 승인하기 전에는 다음을 하지 않는다.

- 원격 push, PR 생성·수정·병합
- 태그 생성·이동·삭제
- GitHub Release 생성·수정·삭제
- 기존 Release asset 삭제·교체·재업로드
- 저장소 설명·Topics·원격 README 수정
- GitHub Actions 수동 실행
- 외부 바이너리·ZIP·진단 자료 업로드
- 실제 사용자에게 배포 또는 공개 링크 생성
- 유료 API 호출
- 컴퓨터 종료·재시작·절전 예약
- 시스템 보안·실행 정책 변경

`dotnet publish`와 ZIP은 로컬 검증용으로만 허용한다. 기존 `v1.0.0`은 어떤 이유로도 수정하지 않는다.

## 2. Git 보호

금지:

- `git reset --hard`
- `git clean -fd`, `git clean -fdx`
- 광범위한 `git checkout -- .`, `git restore .`
- 강제 push, amend, 공개 이력 rebase
- `git add .`, `git add -A`
- 사용자의 untracked 파일 삭제
- 사용자 승인 없는 커밋

필수:

- 시작과 종료 시 현재 브랜치, HEAD, staged·unstaged·untracked 상태 기록
- 기존 변경을 사용자 소유로 보존
- 가능한 경우 Golden Master와 C# 후보를 별도 worktree로 분리
- 현재 작업과 관계없는 파일을 수정하지 않음
- 자동 생성물과 증거를 제품 소스와 구분

## 3. 사용자 데이터와 비밀정보 보호

- 실제 사용자 프로젝트, `%LOCALAPPDATA%`, Workshop 원본, RMK 구독본을 테스트에 쓰거나 수정하지 않는다.
- 익명화 복사본 또는 합성 fixture와 고유한 임시 디렉터리만 사용한다.
- 테스트 입력 원본은 읽기 전용 취급하고, 쓰기 테스트는 별도 복사본에서 수행한다.
- 실제 API 키·토큰·인증 헤더를 사용하지 않는다.
- API 키, 사용자 원문 전체, 개인정보를 로그·fixture·스크린샷·diff·보고서에 남기지 않는다.
- 보안 검사 중 실제 비밀정보를 발견하면 추가 출력 없이 `BLOCKED`로 보고한다.

## 4. 정직한 판정

최종 판정은 다음 중 하나다.

- `PASS`: 자동 필수 게이트를 모두 통과하고 사용자 수동 확인용 RC가 준비됨
- `FAIL`: 실행 가능한 필수 게이트가 실패함
- `BLOCKED`: 환경·권한·서명·법적 권리·수동 UI·깨끗한 PC 등 Codex가 자체 해결할 수 없는 조건

금지:

- 실행하지 않은 검사 PASS 처리
- 빌드 성공을 마이그레이션 완료로 표현
- 화면·메서드 이름만 존재하는 스텁을 기능 완료로 표현
- 테스트를 삭제·skip·완화해 PASS 만들기
- 경고 숨기기, 예외 삼키기, 실패를 성공으로 반환
- “대체로 비슷함”, “문제없어 보임”만으로 판정

## 5. 공개 품질의 정의

### 기능

- Golden Master의 필수 사용자 기능 누락 0건
- 버튼·메뉴·탭·팝업·단축키 동작 누락 0건
- 성공뿐 아니라 취소·실패·재시도·부분 성공·재시작 복구 검증
- 기능 제거·축소로 이전을 단순화하지 않음

### UI와 접근성

- Golden Master를 시각적·기능적 기준으로 사용
- 현재 C# UI를 기준으로 땜질하지 않음
- 컨트롤 누락, 잘림, 겹침과 필수 영역 접근 불가 0건
- 지원 해상도·DPI·테마·최소 창 크기 확인
- 키보드 흐름, 접근성 이름, 포커스 표시 확인
- 최종 사용자 수동 UI 확인 전에는 공개 승인 완료로 표현하지 않음

### 데이터 호환성

- PowerShell판 작업 파일을 C#판에서 그대로 열 수 있음
- 읽기만 했을 때 파일을 자동 변경하지 않음
- 저장 후 번역·상태·메모·이력·경로·설정 보존
- JSON·XML·XLSX·백업에서 사용자 데이터 손실 0건
- 실패한 마이그레이션·저장이 원본을 덮어쓰지 않음
- 손상 파일을 조용히 빈 데이터로 대체하지 않음

### 번역 동등성

- 같은 제공자·모델·프롬프트·용어집·컨텍스트·배치·보호 규칙 유지
- fake HTTP로 canonical request와 response 결과를 비교
- 보호 토큰, XML 태그, 변수, 줄바꿈 손실 0건
- 언어 변경 자체를 번역 품질 향상으로 주장하지 않음

### 보안·개인정보

- API 키·인증정보의 평문 저장과 로그 노출 0건
- XXE, DTD, 경로 이탈, 임의 덮어쓰기, ZIP Slip, 명령 인젝션 방어
- Critical·High 알려진 의존성 취약점 0건
- 관리자 권한을 불필요하게 요구하지 않음
- 원문 외부 전송 대상과 API 키 처리 방법을 사용자 문서에 명시
- 사용자 동의 없는 텔레메트리·원격 전송 0건

### 안정성

- 시작·종료·재시작·강제 종료 후 복구
- 저장 중 실패해도 이전 정상 파일 유지
- 취소·부분 실패 후 반쯤 적용된 상태가 남지 않음
- 백업과 롤백이 실제 작동
- 종료 후 자식 프로세스·백그라운드 작업 0개
- 예외 후 UI 전체가 영구 무응답 상태로 남지 않음

### 성능

- 같은 fixture·환경으로 Golden Master와 비교
- 측정 없는 성능 향상 주장 금지
- 대형 프로젝트에서 UI 장기 정지 없음
- 시작·로드·검색·저장·RMK 처리에 중대한 회귀 없음
- 반복 작업 후 지속적인 메모리 증가 없음

### 유지보수성

- PowerShell 코드를 하나의 거대 Form·Window·클래스로 직역하지 않음
- UI, 상태, 업무 로직, 파일 시스템, HTTP, 저장을 분리
- 핵심 로직이 UI 없이 테스트 가능
- Release 빌드 경고 0을 목표로 하고 미해결 경고는 blocker로 평가
- nullable·분석기 활성화
- 빈 catch, 동기 블로킹, 죽은 코드, 중복 구현 제거
- 현재 규모보다 과도한 추상화는 피함

### PowerShell 0%

최종 후보에서:

- 추적 `.ps1`, `.psm1`, `.psd1` 0개
- runtime의 `powershell.exe`, `pwsh.exe` 호출 0개
- 빌드·테스트·로컬 패키징의 PowerShell 호출 0개
- CI 후보의 PowerShell shell·인라인 명령 0개
- CMD·MSBuild·C# 문자열·리소스를 통한 우회 호출 0개
- `.gitattributes`로 언어 통계만 숨기지 않음

Golden Master는 별도 커밋·worktree와 Git 이력으로 보존한다.

## 6. Release blocker

다음 중 하나라도 있으면 `PASS`를 금지한다.

- 데이터 손실 또는 원본 무단 변경 가능성
- API 키·인증정보 노출
- 경로 이탈·임의 파일 덮어쓰기·XXE·ZIP Slip
- Critical 또는 High 취약점
- 기존 핵심 기능 누락 또는 스텁
- Golden Master와 구조적으로 다른 핵심 UI 흐름
- 사용자 경로의 PowerShell 의존성
- 취소·실패·재시작 후 상태 손상
- 전체 테스트 실패 또는 확인된 flaky test
- 20% 이상의 중대한 성능 회귀가 설명·해결되지 않음
- 로컬 package 실행 실패
- 실제로 수행하지 않은 고위험 검증

수동 UI 확인과 깨끗한 PC 검사가 남은 경우 자동 게이트는 통과할 수 있으나, 공개 단계는 `BLOCKED: manual approval required`로 남긴다.
