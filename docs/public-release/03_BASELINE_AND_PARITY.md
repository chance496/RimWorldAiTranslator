# 03 — Golden Master 기능·UI 기준과 차이 분석

## 목표

사용자가 기능과 화면을 하나씩 다시 설명하지 않아도 되도록 안정판 자체에서 완전한 명세를 만든다. 현재 C# UI나 과거 완료 보고는 기준이 아니다.

## Phase A — Golden Master 실행 가능성

1. Golden Master worktree가 정확히 `4c7d11b49126ba3987e9d49bd16944d4376ba0bc`인지 확인한다.
2. 기존 문서의 정식 실행·테스트 방법을 사용해 실행한다.
3. 테스트 데이터는 별도 임시 경로로 지정한다.
4. 실행에 실제 API가 필요하면 fake/오프라인 경로를 만든다.
5. 시작·종료와 주요 화면 탐색이 가능함을 증거로 남긴다.

기준판이 실행되지 않으면 코드·문서·기존 패키지를 교차 조사한다. 기능을 추측하지 말고 불확실 항목을 `BLOCKED` 또는 `NEEDS_EVIDENCE`로 남긴다.

## Phase B — 기능 인벤토리

최소 조사 범위:

- 최초 실행·초기화·종료
- 프로젝트 생성·열기·전환·삭제·최근 목록
- 모드 검색·경로 선택
- XML 추출·기존 번역 가져오기·원문 변경 감지
- 번역·검토 상태·수동 편집·메모·이력
- 검색·필터·정렬·일괄 작업
- 번역 메모리·용어집·제공자 설정·API 키 취급
- AI 번역·취소·재시도·부분 성공·재개
- 자동 저장·검증·DryRun·적용·백업·롤백
- RMK 탐색·가져오기·XLSX 병합·내보내기
- 진단·로그·테마·키보드 흐름·복구

각 상호작용 요소와 주요 함수에 기능 ID를 부여한다.

```text
PROJ-001 프로젝트 생성
EXTRACT-003 원문 변경 감지
TRANS-004 번역 취소
REVIEW-008 필터 상태 보존
APPLY-005 부분 실패 롤백
RMK-006 XLSX 메타데이터 보존
UI-012 최소 창 크기에서 중지 버튼 접근
```

`FEATURE_PARITY_MATRIX.md` 필수 열:

| ID | 사용자 기능 | 기준 위치 | 입력 | 출력 | 상태 변화 | 오류·취소·재시작 | C# 위치 | 테스트 | 증거 | 판정 |
|---|---|---|---|---|---|---|---|---|---|---|

판정은 `PASS`, `FAIL`, `BLOCKED`, `NOT_IMPLEMENTED`, `NEEDS_EVIDENCE`만 사용한다.

## Phase C — UI 기준 캡처

가능한 조합을 자동화하되 현실적인 지원 범위를 문서화한다.

필수 기준:

- 1366×768, 1600×900, 1920×1080
- 최소 창 크기, 최대화
- DPI 100%, 125%; 가능하면 150%
- 밝은·어두운 테마
- 고대비는 자동 또는 수동 접근성 검사

필수 화면·상태:

- 준비·최초 실행
- 프로젝트 없음·목록·선택·로딩·대시보드
- 번역·검토·설정·용어집·품질·진단·RMK
- 빈 결과·긴 텍스트·한글/영문 혼합
- 소형·5,000행 프로젝트
- 진행·취소·실패·부분 성공·완료
- 확인창·오류창
- 비활성·선택·포커스·hover
- 스크롤 상단·중간·하단

화면별 기록:

- 창·패널 크기와 레이아웃
- 컨트롤 계층·이름·텍스트·기본값
- 폰트·여백·색상·아이콘·테두리
- 탭 순서·접근성 이름·단축키
- 이벤트와 화면 전환 조건
- 동적 영역과 비교 제외 영역

`UI_PARITY_MATRIX.md`에 화면 ID와 상태를 기록한다.

## Phase D — 기준 동작과 데이터 캡처

대표 fixture마다 다음을 남긴다.

- 입력 파일 manifest와 해시
- 시작 상태
- 수행 동작
- 종료 상태와 출력 manifest
- 저장 전후 차이
- 오류·취소 메시지
- API canonical payload
- 성능 기초값

민감한 원문은 해시·길이·구조로 대체한다.

## Phase E — C# 후보 차이 분석

아직 수정하지 말고 다음을 비교한다.

- 기능 ID별 구현·미구현·동작 차이
- 화면 구조·컨트롤·상태 차이
- 데이터 모델과 직렬화 차이
- 프롬프트·배치·재시도 차이
- PowerShell 잔존
- C# 구조의 거대 클래스·UI 결합·동기 블로킹
- 기존 테스트가 실제 Golden Master 동등성을 검증하는지

`GAP_REPORT`를 위험도 순서로 작성한다.

- P0: 데이터 손실·비밀정보·경로 취약점·핵심 기능 부재
- P1: 주요 UI·동작·호환·취소·복구 차이
- P2: 성능·접근성·유지보수성 문제
- P3: 문서·정리·비핵심 차이

## Exit gate

- 모든 주요 UI 상호작용이 기능표에 포함됨
- “용도 불명” 기능 0건 또는 명확한 blocker 기록
- 주요 화면·상태 기준 캡처 완료
- C# 누락과 차이가 증거 기반으로 목록화됨
- 구현 전에 PLAN·DECISIONS·우선순위가 갱신됨
