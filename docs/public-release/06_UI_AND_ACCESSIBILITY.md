# 06 — UI·동작·접근성 동등성

## 1. 기준

- Golden Master UI가 정답이다.
- 현재 C# UI가 더 현대적·간결하다는 이유로 차이를 유지하지 않는다.
- 사용자가 컨트롤을 하나씩 다시 설명하게 하지 않는다.
- 기준 코드, XAML/리소스, 화면과 이벤트를 직접 조사한다.
- 올바른 C# 백엔드는 보존하되 잘못 만든 Form/View/Control은 다시 작성할 수 있다.
- 새 테마·애니메이션·정보 구조·기능은 범위 밖이다.

## 2. 화면 매트릭스

각 화면과 상태에 UI ID를 부여한다.

| UI ID | 화면 | 상태 | 해상도 | DPI | 테마 | 기준 이미지 | 후보 이미지 | 동작 테스트 | 판정 |
|---|---|---|---:|---:|---|---|---|---|---|

필수 조건:

- 1366×768, 1600×900, 1920×1080
- 최소 창 크기, 최대화
- DPI 100%, 125%; 가능하면 150%
- 밝은·어두운 테마
- 고대비 또는 동등한 수동 접근성 검사

필수 상태:

- 최초 실행·준비
- 프로젝트 없음·목록·선택·로딩·대시보드
- 번역·검토·설정·용어집·품질·진단·RMK
- 소형·5,000행 프로젝트
- 빈 검색·긴 텍스트·다국어
- 진행·취소·실패·부분 성공·완료
- 확인·오류 대화상자
- 비활성·선택·포커스·hover
- 스크롤 상단·중간·하단

## 3. 시각 비교

동일 fixture, 창 크기, DPI, 테마, 선택, 스크롤, 텍스트로 촬영한다.

남길 자료:

- baseline 이미지
- candidate 이미지
- 나란히 비교
- 반투명 overlay
- 차이 이미지
- 동적 영역 mask
- 사람이 읽을 차이 설명

픽셀 수치만으로 PASS를 결정하지 않는다. 프레임워크·폰트 렌더링 차이는 고려하되 다음 차이는 허용하지 않는다.

- 컨트롤 누락·추가
- 화면 구조와 작업 순서 변경
- 버튼·탭·메뉴 위치의 의미 있는 변화
- 제목·설명·상태 메시지 누락
- 입력 기본값·선택·활성 상태 차이
- 글자 잘림·겹침·필수 영역 화면 밖 배치
- 팝업 조건과 오류 메시지 흐름 차이

## 4. 동작 동등성

각 상호작용에 자동 또는 수동 테스트를 연결한다.

- 클릭·더블클릭·우클릭
- 탭·단축키·Enter/Escape
- 선택·다중 선택·필터·정렬
- 스크롤 위치와 선택 유지
- 프로젝트 전환과 편집 중 경고
- 저장·자동 저장 표시
- 진행·취소·재시도
- 팝업과 확인창
- 창 닫기·작업 진행 중 종료

Golden Master와 다른 동작이 의도적이면 기능표·DECISIONS와 사용자 승인 근거가 필요하다.

## 5. 접근성

필수:

- 모든 보이는 상호작용 컨트롤의 접근성 이름
- 논리적인 Tab 순서
- 키보드만으로 핵심 흐름 수행
- 포커스 표시
- 색상만으로 상태를 표현하지 않음
- 충분한 텍스트 대비
- 고대비에서 내용 식별
- 최소 창 크기에서 중지·저장 등 필수 버튼 접근
- 스크린리더 기본 탐색 방해 없음
- 오류를 아이콘만이 아니라 텍스트로 설명

## 6. 반응성

- 파일·XML·XLSX·API·대량 필터를 UI 스레드에서 실행하지 않는다.
- 앱 시작 시 반쯤 그려진 UI를 오래 노출하지 않는다.
- 현재 작업과 진행률을 표시한다.
- 취소 버튼이 즉시 시각적으로 반응한다.
- 대형 목록은 virtualization 또는 동등한 최적화를 사용한다.
- 상태 표시가 레이아웃을 밀어내거나 필수 버튼을 숨기지 않는다.
- 창 resize와 테마 전환에서 반복적인 장기 stall이 없어야 한다.

## 7. UI 수정 순서

1. 화면 구조와 누락 컨트롤
2. 사용자 동작과 상태 전이
3. 크기·여백·정렬
4. 글꼴·색상·아이콘·상태 스타일
5. DPI·최소 크기·접근성
6. 성능과 가상화
7. 전체 screenshot 재생성

기존 UI와 차이가 큰 C# 화면을 미세 조정으로 계속 끌고 가지 말고 UI 계층을 기준 구조로 재구축한다.

## Exit gate

- 주요 화면 구조·사용자 흐름 차이 0
- 필수 컨트롤 누락 0
- 글자 잘림·겹침·필수 버튼 접근 불가 0
- 키보드·접근성 필수 항목 통과
- 5,000행에서 장기 UI 정지 없음
- 비교 이미지와 UI matrix 완성
- 자동 PASS와 별도로 사용자 수동 UI 확인 항목 명시
