# 운영 문서 지침

- 문서는 현재 코드와 실행 증거를 반영한다. 추측, 예정 기능, 과거 계획을 구현 완료처럼 쓰지 않는다.
- 코드 근거는 상대 경로와 줄 번호를 기록하고, 줄 번호가 스냅샷 기준임을 밝힌다.
- 현재 상태는 `PROJECT_STATE.md`, 우선순위는 `ROADMAP.md`, 실제 명령은 `QUALITY_GATES.md`, 구조는 `ARCHITECTURE.md`, 장기 결정은 `DECISIONS.md`, 세션 기록은 `WORKLOG.md`에 둔다.
- 작업 기록에 API 키, 사용자 번역 원문 전체, 인증 헤더, 거대한 로그를 넣지 않는다.
- 기능 또는 게이트가 바뀌면 같은 변경에서 관련 문서를 갱신한다.
