# Decisions

## D-001 실제 제품 Git 루트에 운영 체계를 둔다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 제품 문서와 제품 지침은 `tools/RimWorldAiTranslator`에 둔다. 상위 `<workspace>\AGENTS.md`는 작업공간 경계와 공통 안전 규칙만 가진다.
- 이유: 상위 경로는 별도 초기화된 빈 Git 저장소이고 여러 RimWorld 작업이 섞여 있다. 제품 저장소는 독립 `.git`과 GitHub 원격을 가진다.

## D-002 AGENTS에는 상시 규칙만 둔다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 프로젝트 현황, 로드맵, 명령, 구조, 결정과 작업 기록을 `docs/`로 분리한다.
- 이유: 기존 파일은 31,349바이트로 Codex 기본 32,768바이트 한도의 95.7%였다. 상태와 예시가 누적되면 뒤 지침이 잘릴 위험이 있다.

## D-003 범위 제한 지침은 실제 디렉터리에만 둔다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 독립 규칙이 필요한 `native`, `testdata`, `tests`, `docs`에만 추가 `AGENTS.md`를 둔다. v1.0에서 제거된 `launcher` 범위 지침도 함께 제거한다.
- 이유: 존재하지 않는 디렉터리에 지침을 두거나 제품 파일을 옮기는 것은 이번 문서 전용 범위를 넘고 실제 적용 범위도 불명확해진다.

## D-004 상태 문서는 증거 스냅샷이며 실시간 Git이 우선한다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: `PROJECT_STATE.md`의 브랜치, 파일과 줄 번호는 기준 시각을 표시한다. 새 작업은 항상 Git과 코드를 다시 확인한다.
- 이유: 작업 트리와 줄 번호는 바뀌며 오래된 현황이 상시 지침보다 강해지면 안 된다.

## D-005 이번 점검은 무거운 검증을 실행하지 않는다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: PowerShell Parser와 읽기 전용 Git 검사만 수행하고 패키지 재빌드, GUI 실행, 실제 모드/RMK 쓰기와 네트워크 호출은 보류한다.
- 이유: 사용자가 제품 코드 변경을 금지했고, 직전 세션에 호스트 프리즈와 비정상 종료가 있었다. 패키징은 `dist`를 재생성하며 현재 운영 문서 정리에 필수적이지 않다.

## D-006 외부 작업은 현재 요청으로 자동 승인되지 않는다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 로컬 자율 개발은 품질 게이트 안에서 진행할 수 있지만 push, release, 외부 업로드, 유료 API와 시스템 변경은 해당 작업의 명시적 사용자 요청이 필요하다.
- 이유: 로컬 코드 변경과 외부 부작용은 위험과 복구 가능성이 다르다.

## D-007 제공자 capability는 로컬 형식만 검증한다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: URL, HTTPS/루프백, 모델 입력, Temperature와 내장 프로필 일치만 로컬에서 확인한다. 수동 모델 ID는 유지하며 실제 모델 제공 여부는 온라인 미확인으로 표시한다.
- 이유: 모델과 제한은 바뀔 수 있고 실제 조회에는 API 키·네트워크가 필요하다. 추측한 최신값으로 입력을 차단하거나 자동 외부 호출하면 안 된다.

## D-008 번역 메모리는 동일 원문만 비강제로 제안한다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 안전 검사를 통과한 동일 원문 번역만 상태·출처·파일과 함께 용어 탭에 표시하고 번역 칸은 자동 변경하지 않는다.
- 이유: 퍼지 유사도는 RimWorld Def 문맥과 토큰 차이를 놓칠 수 있다. 사용자 번역 자동 덮어쓰기보다 정확한 출처와 명시적 선택이 우선이다.

## D-009 진단 번들은 본문과 식별자를 수집하지 않는다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 진단 ZIP에는 원문·번역문·번역 키·API 키·전체 경로·URL 호스트/쿼리·원시 로그를 넣지 않고 개수·분류·비민감 설정·제품 해시만 넣는다.
- 이유: 지원 편의보다 번역 자산과 자격증명 비노출이 우선이며, 현재 오류의 대부분은 본문 없이 형식·상태·환경·분류로 진단할 수 있다.

## D-010 개척지 작업실 디자인과 실제 상태만 표시한다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 림월드 자산을 복제하지 않는 개척지 작업 관리 분위기를 중앙 테마 토큰으로 구현하고, 번역 진행률은 실제 배치 수가 있을 때만 수치로 표시한다. 알 수 없는 단계는 활동 중 상태와 실제 로그 단계만 보여준다.
- 이유: 제품 정체성과 가독성을 함께 높이면서 가짜 진행률, 장식용 버튼과 화면별 임의 색상을 피하기 위해서다.

## D-011 품질 도구는 검수 데이터를 바꾸지 않고 읽는다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 품질 센터, Diff, 사용량 추정과 보고서는 명시적 편집 상태를 새로 만들거나 번역문을 자동 변경하지 않는다. 5,000행 목록은 가상화하고 검사 결과를 캐시하되 편집 시 무효화한다.
- 이유: 보조 도구가 희소 프로젝트를 비대하게 만들거나 사용자 번역을 바꾸지 않으면서 장시간 검수에 도움을 줘야 한다.

## D-012 장시간 작업 상태는 편집 화면을 가리지 않는다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 원문 분석과 AI 번역은 작업 화면 위의 얇은 상태 막대로 표시한다. 성공·취소는 완료 내용을 남긴 뒤 자동으로 닫고, 사용자 조치가 필요한 오류만 재시도와 닫기 명령을 유지한다.
- 이유: 진행 중에도 기존 번역과 검수 내용을 확인할 수 있어야 하며 정상 완료 때 별도 복귀 조작을 강제하지 않아야 한다.

## D-013 디자인 컨셉과 밝기를 분리한다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 프로페셔널·사이파이·비비드·스튜디오·프런티어 컨셉은 밝음·어두움과 독립적으로 저장한다. 고대비가 켜지면 컨셉보다 접근성 팔레트가 우선한다.
- 이유: 구조와 기능은 유지하면서 사용자가 분위기를 선택하게 하되, 컨셉 때문에 대비와 상태 의미가 손상되지 않게 하기 위해서다.

## D-014 미완성 첫 페인트를 공개하지 않는다

- 날짜: 2026-07-12
- 상태: D-016으로 대체
- 상태: 채택
- 결정: 실행기는 자식 컨트롤이 없는 단일 캔버스 준비 화면을 즉시 표시하고, 40ms 타이머로 상태 문구와 무한 진행 표시 영역만 다시 그린다. WinForms 메인 창은 투명 상태에서 테마·레이아웃·프로젝트 초기화를 끝낸 뒤 한 번에 공개하고, 실행기는 그 공개 상태를 확인한 뒤 준비 화면을 닫는다.
- 이유: 대형 WinForms 화면의 자식 컨트롤이 순서대로 칠해지는 중간 프레임은 기능 오류가 아니어도 제품 완성도를 크게 낮춘다. 전체 창을 이중 버퍼링하면 대형 텍스트·가상 목록 성능과 호환성 위험이 있어 시작 시점만 원자적으로 전환한다.

## D-015 런타임 캐시는 원본 행 밖에 두고 감사 코드는 지연 로드한다

- 날짜: 2026-07-12
- 상태: 채택
- 결정: 검색·미리보기·Def 문맥 런타임 캐시는 `ConditionalWeakTable` 저장소에서 관리하고 프로젝트 전환 시 저장소를 재설정한다. 화면 캡처·접근성·성능 감사 코드는 별도 `UiAudit` 파일로 분리해 감사 옵션에서만 읽는다.
- 이유: JSON 행에 `Add-Member`로 사설 상태를 붙이면 첫 검색 비용과 직렬화·디버깅 혼동이 커진다. 제품 경로에 테스트 전용 코드와 DWM fallback을 항상 파싱할 이유도 없다. 약한 참조는 행 수명과 캐시 수명을 맞추고 원본 데이터를 변경하지 않는다.

## D-016 사용자 런타임을 C#/.NET 8로 통합한다

- 날짜: 2026-07-13
- 상태: 채택
- 결정: 메인 UI, 상태, 추출, 번역, 검수, 적용과 RMK를 C# 프로젝트로 이전하고 win-x64 자체 포함 단일 EXE로 배포한다. 사용자 실행 경로의 PowerShell 실행기와 런타임 스크립트는 제거한다.
- 이유: PowerShell 프로세스 시작·파싱·대형 WinForms 초기화 비용과 실행 정책 의존성을 없애면서, UI와 비즈니스 규칙을 분리하고 취소·회귀 검증을 명시적 타입 경계에 둘 수 있다.
- 호환성: 기존 프로젝트·설정·검수 JSON과 RMK XLSX를 유지한다. PowerShell은 빌드와 용어집 생성 보조 도구에만 남으며 배포 ZIP에는 포함하지 않는다.
