# UI/UX Remediation Plan

기준 시각: 2026-07-12 KST. 이 문서는 P2/P3 재평가의 실행 계약이며 구현 결과는 `PROJECT_STATE.md`와 `WORKLOG.md`에 반영한다.

## 기준 화면과 확인된 문제

기준 캡처 루트: `%TEMP%\RimWorldAiTranslator-ui-remediation-baseline-*`.

| 영역 | 기준선 문제 | 사용자 영향 |
|---|---|---|
| 첫 실행 | 입력 줄과 버튼 뒤에 넓은 빈 공간만 남고 프로그램 목적·첫 행동·작업 순서가 약함 | 처음 쓰는 사용자가 모드 선택과 프로젝트 생성의 차이를 판단하기 어려움 |
| 프로젝트 | 카드가 개별 상태는 보이지만 선택 모드 분석과 다음 행동이 연결되지 않음 | 원문을 먼저 불러와야 하는지, 설정이 준비됐는지 알기 어려움 |
| 설정 | 제공자 상세, 화면 설정과 RMK가 긴 세로 흐름으로 분리됨 | 번역 준비에 필요한 설정을 한눈에 확인하기 어려움 |
| 검수 | 상단 명령 수가 많고 같은 시각 강도로 배치됨 | 장시간 작업에서 현재 상태와 다음 행동 탐색 비용이 큼 |
| 상태 | 작은 진행 막대와 로그 중심이며 로딩·재시도·취소·완료 전환이 약함 | 멈춤과 정상 처리 중을 구분하기 어려움 |
| 품질 | 문제 탭은 요약 텍스트이며 항목 필터·이동·보고서가 없음 | 오류를 찾아 목록과 편집기를 왕복해야 함 |
| 비교 | 역사 탭은 평문 기록으로 기존 번역과 현재 편집의 변화가 강조되지 않음 | 수정 검토 속도와 확신이 떨어짐 |
| 반응형 | 자동 감사상 잘림은 없지만 작은 화면에서 버튼/정보 밀도가 높고 큰 화면에는 빈 공간이 큼 | 해상도에 따라 피로도와 정보 밀도가 크게 달라짐 |

## 디자인 방향

- 독창적인 개척지 운영 터미널: 차콜 셸, 아이보리 작업면, 황토색 주 행동, 구리색 진행, 녹색 성공, 적갈색 오류를 사용한다.
- 색상·텍스트·간격·테두리·버튼·입력·포커스·비활성·진행 상태를 중앙 토큰으로 관리한다.
- 한 화면의 주 행동은 하나만 가장 강하게 표시하고 보조 명령은 절제된 외곽선 스타일로 둔다.
- 프로젝트 → 분석 → 번역 준비 → 실행 → 검수 → 적용의 단계와 현재 위치를 항상 짧은 상태 문구로 보여준다.
- 실제 진행률을 알 수 없으면 백분율 대신 스캔 중인 단계와 활동을 표시한다.

## 구현 항목

1. 공통 디자인 토큰과 앱 셸, 대시보드 빈 상태·최근 프로젝트·준비 상태를 재구성한다.
2. 선택 모드의 이름, ID, 원문 언어와 프로젝트 상태를 실제 카탈로그/프로젝트 데이터로 요약한다.
3. 번역 실행 전 대상·제공자·모델·언어·배치·본문 토큰 추정을 보여주는 사전 점검을 추가한다.
4. 실제 로그와 배치 수에 연결된 개척지 스캔 오버레이로 분석·번역·오류·취소·완료를 표현한다.
5. 품질 검사 센터, 번역 Diff, 결과 보고서와 명령 팔레트를 현재 검수 데이터에 연결한다.
6. 최소/일반/대형 창, 밝음/어두움/고대비, 긴 문장과 각 작업 상태를 캡처하고 1회 이상 수정 반복한다.

## 검증

- PowerShell Parser, 신규 순수 함수 회귀, 전체 20개 회귀와 패키지 빌드.
- 5,000행 UI 감사의 검색/필터/저장 결과 보존과 전후 시작·로드·검색·이동·저장·메모리 비교.
- 빈 상태, 프로젝트, 설정, 사전 점검, 진행, 오류, 취소, 완료, 검수, 품질 센터와 Diff 화면 스냅샷.
- 보이는 상호작용 컨트롤의 접근성 이름, 키보드 포커스, 잘림과 비어 있지 않은 화면 검사.
- 품질 보고서 privacy 검사: API 키, 원문·번역문, 절대 경로 비포함 및 원자 교체/백업 확인.
- 전체 diff의 데이터 무결성·보안·취소·복구·호환성 자체 리뷰.

## 구현 결과

- 공통 테마 토큰, 시작 작업실, 실제 프로젝트 카드, 번역 사전 점검, 개척지 스캔 작업 오버레이, 결과 Diff, 품질 센터, 개인정보 보호 보고서와 명령 팔레트를 실제 데이터 흐름에 연결했다.
- 오버레이는 실제 로그 단계와 배치 수를 해석하며 수치를 알 수 없을 때 가짜 백분율을 표시하지 않는다. 오류에서는 재시도, 취소·완료에서는 검수 프로젝트 복귀와 닫기를 제공한다.
- 품질 센터는 번역 결정을 만들지 않는 읽기 전용 계산과 가상 목록을 사용한다. 5,000행 상태 화면은 초기 구현 21.6초에서 최종 12.7초로 줄었고 내부 품질 계산은 최종 캡처에서 5.6초였다.
- 기준 캡처: `%TEMP%\RimWorldAiTranslator-ui-remediation-baseline-*`.
- 최종 15개 상태 감사: `%TEMP%\RimWorldAiTranslator-ui-remediation-final-*`.
- 캡처 안정화 후 5,000행 품질 화면: `%TEMP%\RimWorldAiTranslator-quality-visual-final-*`.
- 실제 패키지 EXE 시작 화면: `%TEMP%\RimWorldAiTranslator-package-ui-*`.
- 최종 감사는 15/15 상태, 잘림 0건, 보이는 상호작용 컨트롤 접근성 이름 누락 0건을 기록했다. 실제 125/150/200% DPI는 운영체제 표시 배율 변경이 필요해 별도 외부 검증으로 남긴다.
